# テストマトリクス（Draft）

sidecar 必須化後の品質維持を前提に、PDFの多様性に対する検証観点を整理する。

## 対象PDFのカテゴリ

- 2段組（図表少）
- 図表多（Figure/Table密度高）
- 脚注多（Footnotes中心）
- 参考文献形式が異なるもの（著者年式/番号式）
- タイトル/著者ブロックが複数行に分かれるもの

## 合格基準（最低限）

- Markdown本文の可読性が維持される（前回品質から劣化しない）
- 主要見出しが `#` / `##` で構造化される
- 図表キャプションが検出され、本文に残る
- sidecar（`paper.bundle.yaml`）が生成され、リンクラベルが存在する
- `pdf_ref` が欠損しない

## 記録項目

- PDFファイル名 / ページ数
- 主要な崩れ（読み順・段落結合・図表位置）
- QA結果（coverage、sidecar検証）
